# Plan Of Attack

## Context

This is the execution checklist for turning the current Feeling Gamba docs into
the MVP application. The source requirements remain in `mvp-plan.md`, the
durable architecture remains in `architecture/application-architecture.md`, and
the detailed ingestion design remains in `architecture/ingestion-plan.md`.

Use this file to track the order of work. When requirements, architecture,
source assumptions, validation strategy, or app behaviour changes, update the
source-of-truth docs in the same change.

## Key Decisions To Preserve

- The MVP is a reliable historical data log before predictions or betting advice.
- Favourite from pre-race odds and favourite from result-page rank are separate
  concepts.
- MarketMover is only stored when a source explicitly provides the flag.
- Odds snapshots are append-only and remain the source of truth for market state.
- Form Guide is useful for discovery and declared fields, not as the sole market
  source.
- TAB and Betcha GraphQL endpoints are treated as internal web APIs until usage
  rights are confirmed.
- Six-month favourite-performance statistics must not merge result-page
  favourite rank with pre-race favourite snapshots unless the UI labels them as
  separate measures.
